# URI identity

## Requirements

URIs MUST be unique within the combined graph. They SHOULD be stable when names change.

## Recommended schemes

Repositories MAY use resolvable HTTPS URIs or a compact organisational scheme.

Examples:

```text
oea://municipality.example/application/djuma
https://architecture.example.nl/resource/application/djuma
archimate://element-type/application-component
```

## Slugs

Human-readable slugs are recommended, but a URI MUST NOT be treated as a mutable display name.

## Namespaces

A repository SHOULD declare namespace prefixes in `metadata.json` for compact display and export.
