# OpenEA Graph Specification 0.1 — Introduction

## Purpose

OpenEA Graph defines a portable representation for enterprise architecture knowledge. It is designed for small files, human review, version control, automated validation and AI-assisted maintenance.

## Scope

The specification defines:

- resources and relationships;
- resource types and relationship types;
- URI identity and namespaces;
- repository files;
- metamodel packages;
- constraints;
- reasoning rules;
- change sets and provenance.

It does not prescribe a database, user interface, AI model or diagram renderer.

## Normative terminology

The terms **MUST**, **MUST NOT**, **SHOULD**, **SHOULD NOT** and **MAY** indicate requirement levels.
