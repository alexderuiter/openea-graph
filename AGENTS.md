# Instructions for AI agents

When working in this repository:

1. Read `FOUNDATIONS.md` before proposing or making changes.
2. Keep the core small.
3. Prefer plugins, metamodel packages and extensions over core changes.
4. Never introduce inferred facts into the canonical repository. Derived results must remain separate and carry provenance.
5. Preserve backwards compatibility where possible. Describe and justify unavoidable breaking changes.
6. Favor explicitness over magic. Do not hide semantics in filenames, visual styling or undocumented conventions.
7. Keep examples synchronized with the specification and schemas.
8. When adding a feature, first decide whether it belongs in the core or in an extension, and document that decision when it is not obvious.

Keep changes human-readable, Git-friendly and narrowly scoped. AI-generated changes must remain reviewable by a person.
